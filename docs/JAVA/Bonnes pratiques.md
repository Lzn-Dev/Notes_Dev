---------- KISS -----------

Kiss : Keep It Simple and Stupid
=> Toujours garder des choses simple et écrites de la façon la plus bête possible. 
=> Chaque méthode, classe, variable, interface doit avoir des noms aussi claire que possible.
=> La logique à l intérieur des méthodes doit être clair et simple.

- Mettre en oeuvre 
	- Séparer les tâches en plusieurs sous tâches
	- Méthodes courtes > 10 lignes
	- Chaque méthode doit résoudre un petit problème
	- On doit comprendre ce que fait une méthode avec son nom

"Solve the problem, then code it."

----------- DRY -------------

Dry : Don t Repeat Yoursel 
Principe qui consiste à ne pas se répéter. 
Si on se retrouve à écrire plusieurs fois, à plusieurs endroit le même bout de code. On a tout inéret à le placer dans une méthode.
=> Utiliser des classes Helper, créez vos librairies !

"Coder moins, c'est coder mieux."

------------ SOC -------------
SOC : Séparation des préoccupations.
Si une classe fait 2 trucs, fait 2 classes.
Si une méthode fait 2 trucs, fait 2 méthodes.

...
=> Ex : Modèle MVC
